Transmissão de Dados

Quando os dados trafegam, são empacotados em unidades chamadas bits. Esses bits são divididos em pequenos pacotes e transmitidos através de uma rede complexa.

Protocolos

  • TCP: O TCP se preocupa com a integridade dos dados do usuário.
  • UDP: O UDP é mais rápido, porém não é confiável na entrega de dados.

Endereçamento

Os endereços IP são usados para identificar dispositivos em uma rede.
Eles podem ser estáticos ou dinâmicos.
O endereçamento é a forma como os dispositivos em uma rede são identificados e localizados.

URLs

Recurso que serve para a localização de um conteúdo na web.

  • Domínio: exemplo.com
  • Parâmetros: id=123&nome=teste
  • Protocolo: HTTP, HTTPS, FTP, etc.
  • Porta: 80, 443, 21, etc.
  • Caminho: /pasta/arquivo.html

Requisições HTTP

As requisições HTTP são usadas para solicitar recursos de um servidor web.
Existem vários métodos HTTP, incluindo GET, POST, PUT e DELETE.

HTTP (Protocolo de Transferência de Hipertexto) é o protocolo de comunicação utilizado na World Wide Web.

Métodos Principais:

  • GET: Solicita dados do servidor.
  • POST: Envia dados ao servidor, usado para cadastros ou envio de informações.
  • PUT: Atualiza um recurso existente no servidor.
  • DELETE: Remove um recurso específico do servidor.

Características Importantes de HTTP:

  • Envio de Resposta: A resposta é enviada de volta ao cliente pela rede.
  • Body: Contém todos os dados solicitados.
  • Status: 200,400,600
  • Não tem estado: Não guarda a memória de nada
  • Stateless: Cada requisição é independente e não mantém estado entre as interações.
  • Cacheable: Salvar a última informação que você acessou.
  • Layered: Arquitetura em camadas permite a intermediários, como proxies e gateways.
